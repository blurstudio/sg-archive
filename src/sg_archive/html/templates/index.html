You can filter list views by adding various parameters to the url. This is following the format ?key1=value&key2=value2. This is added to the end of the page url.
For example: http://sitename/list_entities/Task?project=1234&sg_status_list=ip&sg_version_type=Camera This example would show the Tasks list page.

  • project=1234 filters to only entities for the project with the id 1234.
  • sg_status_list=ip filters to only entities with their status set to the short code "ip".
  • sg_version_type=Camera filters to only entities with their "Version Type" set to the text "Camera".
If the entity type you are looking at doesn't understand one of the parameters it is ignored. sg_version_type is for the Version entity type and is ignored when looking at tasks. Case is important. The key (sg_status_list) is the code name of a field not its display name. The code name is shown as a tooltip for the
column headercode_name
in the list views. Alternatively you can look at an details page, when looking at the details section it shows the "nice name - code_name" for each field. Use the code_name.

Some common filters:
  • project=1234 Filter by project if the entity type supports it. Use id field
  • sg_status_list=ip The status column. Note that not all entity types use the same status's. Use Short Code field
